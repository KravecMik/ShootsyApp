# Method :: [GET]/Users/CheckUserAccess/[id]/session/[session]/access-to/[needAccessToId]

## Назначение:

Проверка наличия прав пользователя на изменение/удаление записи по заданному идентификатору

## Входные данные:

### Параметры запроса:

```json
{
    "session": "string", //Сессия пользователя
    "needAccessToId": int //Идентификатор, над которым планируется действие. Передается значение UserId в связанных с пользователем сущностях.
}
```

## Пример вызова:

```bash
curl -L -X GET 'http://38.135.55.111:5000/Users/session/8887b2ce-c874-4cd0-adee-74097fe3652a/access-to/2'
```

## Пример ответа:

#### Статус код:
```bash
204
```

## Алгоритм:

1. Получить по указанной сессии пользователя из БД.
2. Проверить у полученного пользователя в таблице user_roles наличие активной записи с ролью суперпользователь.
   2.1 Если роль найдена - вернуть NoContent(204).
3. Если роль не найдена, то проверить совпадает ли идентификатор полученного пользователя с идентификатором для изменения. Если идентификаторы совпадают - вернуть NoContent(204).
   3.1 Если идентификаторы не совпадают - вернуть Forbidden(403).
