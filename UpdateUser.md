# Method :: [PATCH]/Users/UpdateUser/[id]

## Назначение:

Редактирование информации о пользователе

## Входные данные:

### Параметры запроса:

```json
{
    "id": int //Идентификатор пользователя
}
```

### Тело запроса:

```json
[
    {
        "value": "Авдофья",
        "path": "/firstname",
        "op": "replace"
    },
    {
        "path": "/fullname",
        "op": "remove"
    }
]
```

## Пример вызова:

```bash
curl -L -X PATCH 'http://38.135.55.111:5000/Users/2' \
-H 'Content-Type: application/json-patch+json' \
-H 'Session: 8887b2ce-c874-4cd0-adee-74097fe3652a' \
-d '[
    {
        "value": "Авдофья",
        "path": "/firstname",
        "op": "replace"
    },
    {
        "path": "/fullname",
        "op": "remove"
    }
]'
```

## Пример ответа:

```bash
204
```

## Алгоритм:

1. Проверить заполнение обязательных полей. Если поля не заполнены - вернуть BadRequest(400).
2. Проверить наличие и валидность хэдера session в запросе. В случае неудачи - вернуть Unauthorized(401).3. Проверить доступ к редактированию указанного пользователя текущим пользователем, путем проверки наличия роли суперпользователя у редактирующего пользователя или идентичность его идентификатора с редактируемым идентификатором). Если доступа нет - вернуть Forbidden(403).
4. Проверить наличие информации о пользователя в БД. Если запись не найдена - вернуть NotFound(404).
5. Проверить наличие в теле запроса операции, изменяющим поле "Login". Если операция присутствует - вернуть BadRequest(400).
6. Обновить запись в БД согласно введенным данным.
7. Изменить поле EditDate на дату редактирования записи.
7. Вернуть NoContent(204).
