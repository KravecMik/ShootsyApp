# Method :: [POST]/Users/Sign-Up

## Назначение:
Регистрация пользователя в системе

### Аутентификация:
Не требуется

### Описание полей:
|Параметр	|Тип	|Обязательный	|Описание|	Ограничения|
|:---------:|:---------:|:----------:  |:----------------: |:----------:     |
|Login|	string|	Да|	Логин пользователя|	Max: 50 символов|
|Password|	string	|Да	|Пароль пользователя|	Min: 7 символов|
|Firstname|	string|	Да	|Имя пользователя|	Max: 50 символов|
|Lastname|	string|	Нет|	Фамилия пользователя	|Max: 50 символов|
|Description	|string|	Нет|	Описание пользователя	|Max: 250 символов|
|Gender|int	|Да	|Пол пользователя|	См. GenderEnum|
|City|	int|	Да|	Город пользователя|	См. CityEnum|
|Profession|	int|	Да	|IT профессия|	См. ProfessionEnum|

### Пример запроса:
Тело запроса: application/json

```json
{
  "Login": "user123",
  "Password": "password123",
  "Firstname": "Иван",
  "Lastname": "Иванов",
  "Description": "Разработчик с опытом",
  "Gender": 1,
  "City": 3,
  "Profession": 11
}
```

### Возможные коды ответов:
|Код|	Описание|	
|:---------:|:---------:|
|201 Created|	Пользователь успешно создан|
|400 BadRequest|	Ошибка валидации данных|
|400 BadRequest|	Пользователь с таким логином уже существует|

### Пример ответа:
Пример ответа (201 Created)


```json
{
  "userId": 123,
  "session": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

## Алгоритм:
1. Проверить заполнение обязательных полей. Если поля не заполнены - вернуть BadRequest(400).
2. Проверить уникальность поля Login. Если пользователь с таким логином существует в БД - вернуть BadRequest(400).
3. Определить значения CreateDate, EditDate.
4. Сгенерировать соль и хэш пароля пользователя.
5. Создать запись в БД таблице security.users по входным параметрам.
6. На основании идентификатора созданной записи, создать сессию пользователя в БД.
7. Вернуть идентификатор и guid сессии созданного пользователя со статусом Created(201).
