# Method :: [POST]/Users/Sign-In

## Назначение:
Авторизация пользователя в системе

### Аутентификация:
Не требуется

### Описание полей:
|Параметр	|Тип	|Обязательный	|Описание|
|:---------:|:---------:|:----------:  |
|Login|	string|	Да	|Логин пользователя|
|Password	|string|	Да	|Пароль пользователя|


### Пример запроса:
Тело запроса: application/json

```json
{
  "Login": "user123",
  "Password": "password123"
}
```

### Возможные коды ответов:
|Код|	Описание|	 
|:---------:|:---------:|
|200 OK |	Успешная авторизация|
|404 NotFound |	Пользователь не найден|
|400 BadRequest |	Неверный логин или пароль|

### Пример ответа:
Пример ответа (200 OK)

```json
a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

## Алгоритм:
1. Проверить заполнение обязательных полей. Если поля не заполнены - вернуть BadRequest(400).
2. Проверить существование пользователя по логину в БД. Иначе вернуть NotFound(404)
3. Проверить валидность переданных данных. Если данные не валидны - вернуть BadRequest(400) с текстом ошибки: "Неверно указан логин или пароль пользователя"
4. Сгенерировать новую сессию пользователя и вернуть guid сессии со статусом OK(200).
